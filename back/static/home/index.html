Docker y Raspberry Pi: Optimizando Recursos para Contenedores Ligeros

La guía definitiva para exprimir al máximo tus VPS o Raspberry Pi con Docker Compose y límites de memoria.

Publicado el 07 de Diciembre de 2025 | Autor: Lorenzo Carbonell

El Desafío del Hardware Limitado

Como autor del podcast "atareao con Linux", sé que muchos de nosotros, buscando eficiencia y control, montamos nuestros servicios (servidores web, Traefik, bases de datos) en plataformas de bajo consumo como la Raspberry Pi o un VPS pequeño. El problema es: **¿cómo asegurar que Docker no agote nuestros recursos?**

La clave no está solo en usar imágenes ligeras, sino en declarar y limitar explícitamente el uso de RAM y CPU. Esto es especialmente crítico cuando ejecutamos software como **Syncthing** o **PostgreSQL** en contenedores.

Paso 1: Definiendo Límites con Docker Compose

Docker Compose nos ofrece una forma declarativa de gestionar los recursos de nuestros servicios. En lugar de dejar que un contenedor consuma toda la memoria disponible, podemos establecer un límite estricto (`limits`) y una reserva blanda (`reservations`).

Ejemplo de `docker-compose.yml` optimizado:

Aquí definimos límites para un contenedor web simple que no debería exceder los 128MB de RAM:

version: "3.7"
services:
  nginx-web:
    image: nginx:stable-alpine
    restart: always
    ports:
      - "80:80"
    deploy:
      resources:
        limits:
          memory: 128M
          cpus: '0.50' # Limita el uso a medio núcleo
        reservations:
          memory: 32M # Mínimo que siempre tendrá disponible

Utilizar la imagen `alpine` garantiza que estamos partiendo de la base más pequeña posible, optimizando el uso del disco y el inicio.

Paso 2: Optimizaciones Adicionales para la Productividad

"En Linux, la productividad no se trata solo de la velocidad, sino de la eficiencia de los recursos que administras."

Más allá de Docker, siempre recomiendo revisar estos puntos en tu sistema Linux base:

  • **Deshabilitar servicios innecesarios:** Si estás usando un VPS sin interfaz gráfica, asegúrate de deshabilitar cualquier servicio relacionado con el escritorio (como extensiones de **GNOME** o servicios gráficos).
  • **Limpieza del sistema:** Utiliza `docker system prune` regularmente para eliminar imágenes, contenedores y volúmenes sin usar.
  • **Compilación nativa:** Si estás trabajando con lenguajes como **Rust**, compila binarios específicamente para la arquitectura ARM de la Raspberry Pi para obtener mejor rendimiento nativo.
